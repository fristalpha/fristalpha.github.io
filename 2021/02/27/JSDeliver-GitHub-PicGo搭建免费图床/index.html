JSDeliver+GitHub+PicGo搭建免费图床

发表于2021-02-27|更新于2021-03-10|教程
|字数总计:695|阅读时长:2分钟|阅读量:|评论数:

概念简介

图床

图床一般是指储存图片的服务器，有国内和国外之分。国外的图床由于有空间距离等因素决定访问速度很慢影响图片显示速度。国内也分为单线空间、多线空间和cdn加速三种。—百度百科

CDN

CDN的全称是Content Delivery Network，即内容分发网络。CDN是构建在现有网络基础之上的智能虚拟网络，依靠部署在各地的边缘服务器，通过中心平台的负载均衡、内容分发、调度等功能模块，使用户就近获取所需内容，降低网络拥塞，提高用户访问响应速度和命中率。CDN的关键技术主要有内容存储和分发技术。—百度百科

JSDelive

jsDelivr是由Dmitriy Akulov和Prospect One开发的公共，开源CDN（内容交付网络），专注于性能，可靠性和安全性。每个人都可以免费使用，没有带宽限制。—JSD官网

GitHub

GitHub是一个面向开源及私有软件项目的托管平台，因为只支持Git作为唯一的版本库格式进行托管，故名GitHub。—百度百科

PicGo

PicGo: 一个用于快速上传图片并获取图片 URL 链接的工具 。—PicGo官方文档

开始搭建

新建图床仓库

登录自己的GitHub，点击右上方的加号，选择new repository新建一个仓库，作为图床仓库。

新建 一个访问令牌

点击右上角的头像，选择settings—>Developer settings—>Personal access tokens,然后点击Generate new token，勾选repo项，然后点击Generate token生成一个token。

然后把这个token复制下来，这里这个token只会出现一次，在后面配置PicGo时要用到。

配置PicGo

下载地址：GitHub

安装完成后打开软件，选择图床设置—>GitHub图床

设定仓库名：用户名/图床仓库名

设定分支名：master

设定Token：刚复制的那个

指定储存路径：在你仓库里存放图片的路径

设定自定义域名：你的图片的访问链接，由于图片是上传到GitHub仓库中的，所以图片路径为https://github.com/用户名/仓库名/文件夹名/图片名，但我们还需要通过JSDelivr来加速图片（ jsDelivr 提供加速 GitHub 仓库的域名 cdn.jsdelivr.net/gh ），所以此处应填写成

使用

点击上传区，把图片拖到指定区域即可将文件上传至GitHub图床仓库指定目录下。

然后点击相册即可使用。
